Streaming SQL & Windows
Run continuous Flink SQL with tumbling, hopping and session windows.
Results stream live into the Results tab as data arrives.
Tumble window (1 min buckets):
SELECT window_start, COUNT(*) AS cnt
FROM TABLE(TUMBLE(TABLE orders,
  DESCRIPTOR(event_time), INTERVAL '1' MINUTE))
GROUP BY window_start;

Ctrl+Enter to run · Ctrl+/ to comment · Ctrl+S to save

UDF Functions
Extend Flink SQL with custom logic — scalar, table, aggregate and async.
SQL UDFs register instantly with no JAR needed.
SQL UDF — no JAR required:
CREATE TEMPORARY FUNCTION classify(score DOUBLE)
RETURNS STRING LANGUAGE SQL AS $$
  CASE WHEN score >= 0.8 THEN 'HIGH'
      WHEN score >= 0.5 THEN 'MED'
      ELSE 'LOW' END
$$;

Open ⨍ UDFs → SQL UDF for a guided form with live preview.

Amazon Managed Service for Apache Flink
No Flink SQL Gateway REST API. Uses AWS KinesisAnalyticsV2 API + IAM auth.
Interactive SQL runs inside Studio notebooks (Apache Zeppelin).
Studio notebooks use Apache Zeppelin — write Flink SQL with %flink.ssql paragraphs. Open via AWS Console → your application → Open in Apache Zeppelin.
Flink applications (Java/Python/Scala) are deployed via AWS CLI or SDK. If you've port-forwarded the Flink REST UI (:8081), enter its URL below then use Direct Gateway mode.
Found in: AWS Console → Managed Flink → your app → Open in Apache Zeppelin. App must be RUNNING.
🔑 AWS IAM Credentials
Uses AWS_ACCESS_KEY_ID / AWS_SECRET_ACCESS_KEY env vars, ~/.aws/credentials, or EC2/ECS instance role. No input needed — AWS SDK picks these up automatically.
Assumes this role via STS. The role must have kinesisanalyticsv2:* permissions.
⚠ No Flink SQL Gateway — SQL execution in Str:::lab Studio is not available for AWS Managed Flink via the browser.
Options: Use the Studio notebook in the AWS Console · Port-forward :8081 then use Direct Gateway · Run a self-managed SQL Gateway in the same VPC.
