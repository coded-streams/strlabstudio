Register a Connector in
Systems Manager → Use in Pipeline

Learn the full end-to-end connector workflow. Register a PostgreSQL connection in Systems Manager, test its reachability, save it, then build a pipeline in Pipeline Manager where the saved connector automatically prefills the JDBC sink node — no re-typing connection details.

✓ Flink session connected ⚠ JDBC JAR + PG driver JAR ⚠ PostgreSQL running
This Demo What You'll Build Prerequisites Part A — Systems Manager 1 · Upload JDBC JARs 2 · Create PG Table 3 · Register Integration 4 · Test Connectivity 5 · Save Integration Part B — Pipeline Manager 6 · Open Pipeline 7 · Datagen Source 8 · Filter Transform 9 · JDBC Sink (prefill!) 10 · Print Sink 11 · Connect & Submit 12 · Verify Reference Generated SQL

▦ What You'll Build

This demo has two parts. In Part A, you register a PostgreSQL integration in Systems Manager — enter host, port, credentials, test connectivity, and save. In Part B, you build a pipeline. When you open the JDBC Sink node's config, the saved integration appears as a banner with a Use button that fills all connection fields automatically.

ℹ
Why save connectors first? In real projects you have many pipelines writing to the same database. Saving the connection once means every new JDBC sink in any pipeline can reuse it with one click — no copy-pasting JDBC URLs or passwords.

! Prerequisites

  • JDBC JAR — flink-connector-jdbc-x.x.x.jar in /opt/flink/lib/
  • PostgreSQL driver — postgresql-42.x.x.jar in /opt/flink/lib/
  • PostgreSQL instance — reachable at postgres:5432 (update to your host)
⚠
Restart the SQL Gateway after copying JARs into /opt/flink/lib/. The JDBC connector cannot be loaded with ADD JAR at runtime — it must be present at startup.

A Part A — Systems Manager

1

Upload the JDBC JARs

Click ⊙ Systems in the topbar → ⬆ Upload JAR tab. Use the download links on that page to get:

  • flink-connector-jdbc-3.2.0-1.19.jar — from Maven Central
  • postgresql-42.7.3.jar — from Maven Central

Upload each one using the dropzone. Then restart the Gateway container:

docker restart flink-sql-gateway

Reconnect your Studio session after the restart.

2

Create the target table in PostgreSQL

Before Flink can write to PostgreSQL, the table must already exist. Run this in psql or your PostgreSQL client:

CREATE TABLE IF NOT EXISTS flink_orders (
  order_id  BIGINT       PRIMARY KEY,
  customer  VARCHAR(200),
  amount    DOUBLE PRECISION,
  status    VARCHAR(50),
  ts        TIMESTAMP
);
3

Register the PostgreSQL integration

Click ⊙ Systems → ⊙ Integrations tab → click the PostgreSQL card. The integration form opens on the right. Fill in:

Integration Namepostgres_local
Hostpostgres
Port5432

Auth ModeUsername / Password
Usernameflink_user
Passwordsecret

The Generated Flink SQL panel below the form updates in real time to show the CREATE TABLE ... WITH ('connector'='jdbc', ...) statement that will be used.

4

Test Connectivity

Click the ⊙ Test Connectivity button. You should see:

✓
Flink cluster reachable ✓ — PostgreSQL probe sent
The test confirms the Flink cluster is up and gives you the exact command to verify PostgreSQL reachability from inside the Flink container network:
docker exec <flink-container> bash -c "nc -zv postgres 5432 && echo OPEN"

Run the nc -zv command from your terminal. If it prints OPEN, PostgreSQL is reachable from Flink.

5

Save the Integration

Click 💾 Save Integration. A success toast appears. The integration is saved to browser localStorage. Click the 💾 Saved tab to confirm postgres_local appears in the list.

💡
Now any JDBC Sink node in Pipeline Manager will show this connector. When you open a JDBC Sink config modal, a green banner shows saved matching integrations. Click Use to fill all fields in one click.

B Part B — Pipeline Manager

6

Open Pipeline Manager

Click ◈ Pipeline. Rename the pipeline to pg-sink-demo. Set checkpointing in ⚙ Settings:

Execution Mode:      streaming
Parallelism:         1
Checkpoint Interval: 10000
7

Add Datagen source

Drag Sources → Datagen. Click ✏ edit:

Rows / sec5

8

Add Filter transform

Drag Transformations → Filter. Click ✏ edit. The Filter node has one parameter only — the WHERE condition. There is no Table Name field on Filter.

Filter
Node Labelfilter_orders
↑ canvas label only
WHERE Conditionamount > 100.0

Click ✓ Apply. This becomes WHERE amount > 100.0 in the INSERT INTO.

9

Add JDBC Sink — use the saved connector!

Drag Sinks → JDBC Sink. Click ✏ edit. At the top of the modal you will see the connector banner:

✓ Connector detected
postgres_local Use

Click Use. All JDBC fields are filled automatically from the saved integration. Then complete the remaining fields:

Click ⊙ Test Connectivity to verify PostgreSQL is still reachable. Then click ✓ Apply.

10

Add Print (Debug) sink

Drag Sinks → Print (Debug). Click ✏ edit:

Print Sink
Table Nameorders_debug

Click ✓ Apply.

11

Connect nodes, Layout, Submit

Draw these edges:

  • orders_stream → filter_orders — FORWARD
  • filter_orders → pg_orders_sink — HASH
  • filter_orders → orders_debug — FORWARD

Click Layout → ▶ Run to preview animation → Submit.

12

Verify the output

-- In psql — rows should appear within seconds:
SELECT order_id, customer, amount, status
FROM flink_orders
ORDER BY ts DESC
LIMIT 10;

-- Count total rows inserted so far:
SELECT COUNT(*) FROM flink_orders;

-- Check print debug in TaskManager log:
docker logs flink-taskmanager --tail=20

⟨⟩ Generated SQL

SET 'execution.runtime-mode'           = 'streaming';
SET 'parallelism.default'              = '1';
SET 'execution.checkpointing.interval' = '10000';

CREATE TABLE IF NOT EXISTS orders_stream (
  order_id  BIGINT,
  customer  STRING,
  amount    DOUBLE,
  status    STRING,
  ts        TIMESTAMP(3)
) WITH (
  'connector'       = 'datagen',
  'rows-per-second' = '5'
);

CREATE TABLE IF NOT EXISTS pg_orders_sink (
  order_id  BIGINT,
  customer  STRING,
  amount    DOUBLE,
  status    STRING,
  ts        TIMESTAMP(3),
  PRIMARY KEY (order_id) NOT ENFORCED
) WITH (
  'connector'  = 'jdbc',
  'url'        = 'jdbc:postgresql://postgres:5432/mydb',
  'table-name' = 'public.flink_orders',
  'username'   = 'flink_user',
  'password'   = 'secret'
);

CREATE TABLE IF NOT EXISTS orders_debug (
  order_id  BIGINT,
  customer  STRING,
  amount    DOUBLE,
  status    STRING,
  ts        TIMESTAMP(3)
) WITH (
  'connector' = 'print'
);

-- Fan-out: JDBC sink gets filtered orders
INSERT INTO pg_orders_sink
SELECT * FROM orders_stream WHERE amount > 100.0;

-- Print sink also gets filtered orders
INSERT INTO orders_debug
SELECT * FROM orders_stream WHERE amount > 100.0;
