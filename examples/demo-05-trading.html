Real-Time Trading Analytics
— 4 Pipelines, 5 Kafka Topics

A complete end-to-end production-grade streaming architecture built entirely in Pipeline Manager. Datagen produces synthetic trade events into Kafka. Three consumer pipelines read from the same enriched topic and fan out to: an alerts sink for high-value trades, a 1-minute OHLCV window aggregation, and an exchange routing breakdown. A fifth pipeline provides a live preview in the Results tab.

▦ Architecture Overview

Five separate pipelines run as independent Flink jobs. They are connected through Kafka topics — not through Flink directly. This is the production pattern: each job can be scaled, restarted, or upgraded without affecting the others.

Concepts practised: Kafka source and sink configuration, chained producer → consumer pipelines, Filter node, Project node with CASE WHEN expressions, Tumble Window with aggregations, edge types, multiple independent pipelines sharing a Kafka topic, live preview from a reader table.

! Prerequisites

  • Kafka JAR — flink-sql-connector-kafka-x.x.x.jar in /opt/flink/lib/. Use ⊙ Systems → Upload JAR if not yet installed.
  • Five Kafka topics created before any pipeline runs — see the next step.
  • Studio connected — active Flink session in the topbar.
  • Run session settings first — the SET statements must be run once per session before any pipeline is submitted.
⚠
Run in order. Pipeline A must be running and producing data before Pipeline B is submitted. Pipeline B must be running before Pipelines C (Jobs 3, 4, 5) are submitted. Allow ~15–30 seconds between each before submitting the next.

0 Create Kafka Topics

Before building any pipeline, create the five Kafka topics. Run this command from your terminal (replace kafka-01 with your Kafka container name):

docker exec -it kafka-01 bash -c "
  for topic in demo.trades.raw demo.trades.enriched \
               demo.trades.summary demo.alerts.high demo.trades.byexch; do
    kafka-topics.sh --bootstrap-server localhost:9092 --create \
      --topic \$topic --partitions 4 --replication-factor 1
  done
  echo 'Topics created'"

Verify they exist:

docker exec -it kafka-01 kafka-topics.sh \
  --bootstrap-server localhost:9092 --list | grep demo

You should see all five topics listed.

⚙ Session Settings

Open a new tab in Studio and run these SET statements once per session before submitting any pipeline. These configure checkpointing and performance optimisations that apply to all pipelines in this session.

USE CATALOG default_catalog;
USE `default`;

SET 'execution.runtime-mode'                   = 'streaming';
SET 'parallelism.default'                      = '2';
SET 'pipeline.operator-chaining'               = 'false';
SET 'execution.checkpointing.interval'         = '15000';
SET 'execution.checkpointing.mode'             = 'EXACTLY_ONCE';
SET 'execution.checkpointing.timeout'          = '60000';
SET 'execution.checkpointing.min-pause'        = '5000';
SET 'table.exec.state.ttl'                     = '3600000';
SET 'table.exec.source.idle-timeout'           = '10000';
SET 'table.exec.mini-batch.enabled'            = 'true';
SET 'table.exec.mini-batch.allow-latency'      = '2000 ms';
SET 'table.exec.mini-batch.size'               = '500';
💡
Shortcut: these SET statements are available as a snippet in Studio. Click Snippets → ⚡ Recommended Streaming Config and they will be inserted into the current tab automatically.

▶ Live Preview — Results Tab

To see enriched trade data streaming live in the Results tab, create a preview reader table with a separate consumer group and run a SELECT. Do this in a Studio SQL editor tab — not in Pipeline Manager.

-- Create a reader table (won't interfere with the pipeline consumer groups)
CREATE TEMPORARY TABLE enriched_preview (
  trade_id    STRING,
  user_id     STRING,
  symbol      STRING,
  side        STRING,
  quantity    DOUBLE,
  price       DOUBLE,
  exchange    STRING,
  category    STRING,
  trade_value DOUBLE,
  risk_level  STRING,
  event_time  TIMESTAMP(3),
  WATERMARK FOR event_time AS event_time - INTERVAL '5' SECOND
) WITH (
  'connector'                    = 'kafka',
  'topic'                        = 'demo.trades.enriched',
  'properties.bootstrap.servers' = 'kafka-01:29092',
  'properties.group.id'          = 'studio-preview-consumer',
  'scan.startup.mode'            = 'earliest-offset',
  'format'                       = 'json'
);

-- Stream rows into the Results tab:
SELECT
  trade_id,
  symbol,
  side,
  ROUND(quantity, 2)   AS qty,
  ROUND(price, 2)      AS price,
  trade_value,
  risk_level,
  exchange,
  event_time
FROM enriched_preview;
💡
Use Search rows + Colour Describe. Type CRITICAL in the Search rows box to see only high-value trades. Then open 🎨 Colour Describe → add a rule on the risk_level column: == CRITICAL → red background, == HIGH → yellow background, == LOW → green background. Rows will highlight in real time as they stream in.

⟨⟩ All Generated SQL

CREATE TABLE IF NOT EXISTS trade_generator ( ... )
WITH ('connector'='datagen','rows-per-second'='10');

CREATE TABLE IF NOT EXISTS raw_trades_sink (
  trade_id STRING, user_id STRING, symbol STRING, side STRING,
  quantity DOUBLE, price DOUBLE, exchange STRING,
  category STRING, event_time TIMESTAMP(3)
) WITH (
  'connector'                    = 'kafka',
  'topic'                        = 'demo.trades.raw',
  'properties.bootstrap.servers' = 'kafka-01:29092',
  'format'                       = 'json'
);

INSERT INTO raw_trades_sink
SELECT
  trade_id, user_id,
  CASE symbol_id WHEN 1 THEN 'BTC' WHEN 2 THEN 'ETH'
    WHEN 3 THEN 'SOL' WHEN 4 THEN 'AAPL'
    WHEN 5 THEN 'TSLA' ELSE 'NVDA' END AS symbol,
  CASE side_id WHEN 1 THEN 'BUY' ELSE 'SELL' END AS side,
  quantity, price,
  CASE exchange_id WHEN 1 THEN 'BINANCE' WHEN 2 THEN 'COINBASE'
    WHEN 3 THEN 'NYSE' ELSE 'NASDAQ' END AS exchange,
  CASE category_id WHEN 1 THEN 'CRYPTO'
    WHEN 2 THEN 'EQUITY' ELSE 'ETF' END AS category,
  event_time
FROM trade_generator;
-- Pipeline B: Raw → Enriched
INSERT INTO enriched_sink
SELECT
  trade_id, user_id, symbol, side, quantity, price,
  exchange, category,
  ROUND(quantity * price, 2) AS trade_value,
  CASE
    WHEN quantity * price > 100000 THEN 'CRITICAL'
    WHEN quantity * price > 10000  THEN 'HIGH'
    WHEN quantity * price > 1000   THEN 'MEDIUM'
    ELSE 'LOW'
  END AS risk_level,
  event_time
FROM raw_trades_source;
-- Pipeline C1: High-value trade alerts
INSERT INTO high_alerts_sink
SELECT *
FROM enriched_source
WHERE trade_value > 50000;
-- Pipeline C2: 1-minute OHLCV window
INSERT INTO trade_summary_sink
SELECT
  window_start, window_end, symbol, side,
  COUNT(*)                                         AS trade_count,
  ROUND(SUM(quantity), 4)                          AS total_volume,
  ROUND(SUM(trade_value), 2)                       AS total_value,
  ROUND(AVG(price), 2)                             AS avg_price,
  MIN(price)                                       AS min_price,
  MAX(price)                                       AS max_price,
  SUM(CASE WHEN trade_value > 10000 THEN 1 ELSE 0 END) AS high_risk_count
FROM TABLE(TUMBLE(TABLE enriched_source,
  DESCRIPTOR(event_time), INTERVAL '1' MINUTE))
GROUP BY window_start, window_end, symbol, side;
-- Pipeline C3: Exchange breakdown
INSERT INTO by_exchange_sink
SELECT
  trade_id, user_id, symbol, side, exchange,
  risk_level, trade_value,
  trade_value > 50000 AS is_large_trade,
  event_time
FROM enriched_source;
