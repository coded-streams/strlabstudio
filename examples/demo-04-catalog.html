Create a PostgreSQL Catalog
→ Build a Pipeline Using It

Register a persistent JDBC catalog backed by PostgreSQL using Catalog Manager. Browse the live table list in the sidebar. Then build a pipeline that reads from a Datagen source, filters and enriches events, and writes to a catalog-managed PostgreSQL table — with live verification from the sidebar.

▦ What You'll Build

After this demo you will understand the difference between a Flink connector (how data moves) and a Flink catalog (where table metadata lives). The catalog makes PostgreSQL tables visible in the Studio sidebar — click any column name to insert it into the SQL editor. The pipeline writes enriched, classified events directly into the catalog-registered table.

📦 pg_catalog ← registered by Catalog Manager
🗄 public
📋 app_events ← pre-existing PG table
event_id BIGINT
service VARCHAR
severity INT
ts TIMESTAMP
📋 enriched_events ← sink table for this pipeline
⊕
Catalog vs Connector The catalog stores the schema definition of enriched_events. The JDBC connector is the runtime mechanism that writes rows to it. You need both — but with a catalog registered, Flink knows the schema without you defining it in the pipeline node.

! Prerequisites

  • JDBC JAR — flink-connector-jdbc-x.x.x.jar in /opt/flink/lib/ (see Demo 3 Step 1 for how to upload)
  • PostgreSQL driver — postgresql-42.x.x.jar in /opt/flink/lib/
  • PostgreSQL instance — reachable at postgres:5432
  • Both JARs installed from Demo 3? Skip to Step 1 — no need to re-upload.

A Part A — Create the Catalog

1

Create tables in PostgreSQL first

The catalog will read these table definitions from PostgreSQL's information_schema. Run this in psql:

-- Source reference table (we'll generate data synthetically, not read from this)
CREATE TABLE IF NOT EXISTS app_events (
  event_id   BIGINT       PRIMARY KEY,
  service    VARCHAR(100),
  message    VARCHAR(500),
  level      VARCHAR(20),
  event_ts   TIMESTAMP
);

-- Sink table — Flink will write enriched rows here
CREATE TABLE IF NOT EXISTS enriched_events (
  event_id       BIGINT       PRIMARY KEY,
  service        VARCHAR(100),
  message        VARCHAR(500),
  level          VARCHAR(20),
  severity_score INT,
  event_ts       TIMESTAMP
);
2

Open Catalog Manager

Click ⊕ Catalogs in the Studio topbar. The Catalog Manager opens on the ⊕ Create Catalog tab. You will see a grid of catalog types. Click PostgreSQL (JDBC).

The form expands below the type picker. A yellow banner warns you that the JDBC JARs must be in /opt/flink/lib/.

3

4

Test Connectivity

Click ⊙ Test Connectivity. Expected result:

✓
Flink cluster reachable ✓ — PostgreSQL probe sent
The result also shows the nc -zv command. Run it from your terminal to confirm PostgreSQL is reachable from inside the Flink container:
docker exec flink-sql-gateway bash -c "nc -zv postgres 5432 && echo OPEN"
5

Create the Catalog

Click ⚡ Create Catalog. Studio executes:

CREATE CATALOG pg_catalog WITH (
  'type'             = 'jdbc',
  'default-database' = 'public',
  'username'         = 'flink_user',
  'password'         = 'secret',
  'base-url'         = 'jdbc:postgresql://postgres:5432/mydb'
);
USE CATALOG pg_catalog;

A green success message appears. Because you set Switch To After: Yes, the active catalog is now pg_catalog. The status bar at the bottom of Studio updates to catalog: pg_catalog.public.

6

Browse the catalog in the sidebar

In the Studio left sidebar, click the CATALOG tab. Click ⟳ Catalog in the topbar to refresh. Expand the tree:

▶ pg_catalog
▶ public
📋 app_events
📋 enriched_events

Click enriched_events to expand it. All column names and types appear. Click any column name — it inserts at the cursor position in the SQL editor.

💡
This is live. Any table you create in PostgreSQL while the session is running will appear here after a ⟳ refresh. The catalog is not a snapshot — it queries information_schema in real time.

B Part B — Build the Pipeline

7

Open Pipeline Manager

Click ◈ Pipeline. Rename to catalog-pipeline. In ⚙ Settings:

Execution Mode:      streaming
Parallelism:         1
Checkpoint Interval: 10000
8

Add Datagen source

Drag Sources → Datagen. Click ✏ edit:

Datagen Source
Table Nameraw_events_gen
Rows / sec5
Schemaevent_id BIGINT service STRING message STRING level STRING event_ts TIMESTAMP(3)

Click ✓ Apply.

9

Add Filter — keep only WARNING and ERROR

Drag Transformations → Filter. Click ✏ edit. Filter has one parameter only — the WHERE condition. There is no Table Name field.

Filter
Node Labelfilter_important
↑ canvas label only
WHERE Conditionlevel = 'ERROR' OR level = 'WARNING'

Click ✓ Apply.

10

Add Project — compute severity_score

Drag Transformations → Project. Click ✏ edit. Project has one parameter — the SELECT expressions, one per line. No Table Name field.

Project
Node Labelproject_enrich
↑ canvas label only
SELECT Expressions
(one per line)event_id, service, message, level, CASE WHEN level = 'ERROR' THEN 10 WHEN level = 'WARNING' THEN 5 ELSE 1 END AS severity_score, event_ts

Click ✓ Apply. These become the SELECT column list in the INSERT INTO.

11

Add JDBC Sink — write to the catalog table

Drag Sinks → JDBC Sink. Click ✏ edit:

JDBC Sink
Table Namepg_enriched_sink
JDBC URLjdbc:postgresql://postgres:5432/mydb
DB Tablepg_catalog.public.enriched_events
Usernameflink_user
Passwordsecret
Schemaevent_id BIGINT service STRING message STRING level STRING severity_score INT event_ts TIMESTAMP(3)
⊕
Notice the DB Table value: pg_catalog.public.enriched_events uses the three-part catalog reference. Flink resolves the schema from the catalog — your node schema definition must match the PostgreSQL table columns.

Click ⊙ Test Connectivity, then ✓ Apply.

12

Add Print (Debug) sink

Drag Sinks → Print (Debug). Click ✏ edit:

Print Sink

Click ✓ Apply.

13

Connect nodes, Layout, Submit

Draw these edges in order:

  • raw_events_gen → filter_important — FORWARD
  • filter_important → project_enrich — FORWARD
  • project_enrich → pg_enriched_sink — HASH
  • project_enrich → events_debug — FORWARD

Click Layout → ▶ Run (check animation looks right) → Submit.

✓ Verify in PostgreSQL and the Sidebar

-- In psql — rows should appear within 10 seconds:
SELECT event_id, service, level, severity_score, event_ts
FROM enriched_events
ORDER BY event_ts DESC
LIMIT 10;

-- Only ERROR (score=10) and WARNING (score=5) rows:
SELECT level, severity_score, COUNT(*) AS total
FROM enriched_events
GROUP BY level, severity_score;

-- Expected:
--  level   | severity_score | total
-- ---------+----------------+-------
--  ERROR   |             10 |   ...
--  WARNING |              5 |   ...

From Studio, run these queries using the catalog:

-- Make sure we are in the catalog context:
USE CATALOG pg_catalog;
USE public;

-- Query the sink table directly through the catalog:
SELECT level, severity_score, service
FROM enriched_events
ORDER BY event_ts DESC
LIMIT 20;

-- The catalog sidebar lets you click column names to insert them.
-- Try: click "severity_score" in the sidebar tree —
-- it inserts at your cursor position in the editor.
💡
No CREATE TABLE needed. Because pg_catalog is registered, you can SELECT from enriched_events directly — Flink reads the schema from the catalog. There is no need to write any DDL in the SQL editor.
-- Check TaskManager log for print debug output:
docker logs flink-taskmanager --follow --tail=20

-- Rows look like (event_id, service, message, level, severity_score, ts):
-- +I[1234, svc-auth, ..., ERROR, 10, 2026-...]
-- +I[5678, svc-api,  ..., WARNING, 5, 2026-...]

⟨⟩ Generated SQL

-- Generated by Catalog Manager:
CREATE CATALOG pg_catalog WITH (
  'type'             = 'jdbc',
  'default-database' = 'public',
  'username'         = 'flink_user',
  'password'         = 'secret',
  'base-url'         = 'jdbc:postgresql://postgres:5432/mydb'
);
USE CATALOG pg_catalog;
-- Generated by Pipeline Manager on Submit:
SET 'execution.runtime-mode'           = 'streaming';
SET 'parallelism.default'              = '1';
SET 'execution.checkpointing.interval' = '10000';

CREATE TABLE IF NOT EXISTS raw_events_gen (
  event_id  BIGINT,
  service   STRING,
  message   STRING,
  level     STRING,
  event_ts  TIMESTAMP(3)
) WITH (
  'connector'       = 'datagen',
  'rows-per-second' = '5'
);

CREATE TABLE IF NOT EXISTS pg_enriched_sink (
  event_id        BIGINT,
  service         STRING,
  message         STRING,
  level           STRING,
  severity_score  INT,
  event_ts        TIMESTAMP(3),
  PRIMARY KEY (event_id) NOT ENFORCED
) WITH (
  'connector'  = 'jdbc',
  'url'        = 'jdbc:postgresql://postgres:5432/mydb',
  'table-name' = 'pg_catalog.public.enriched_events',
  'username'   = 'flink_user',
  'password'   = 'secret'
);

CREATE TABLE IF NOT EXISTS events_debug (
  event_id        BIGINT,
  service         STRING,
  message         STRING,
  level           STRING,
  severity_score  INT,
  event_ts        TIMESTAMP(3)
) WITH ( 'connector' = 'print' );

INSERT INTO pg_enriched_sink
SELECT
  event_id, service, message, level,
  CASE
    WHEN level = 'ERROR'   THEN 10
    WHEN level = 'WARNING' THEN  5
    ELSE 1
  END AS severity_score,
  event_ts
FROM raw_events_gen
WHERE level = 'ERROR' OR level = 'WARNING';

INSERT INTO events_debug
SELECT
  event_id, service, message, level,
  CASE
    WHEN level = 'ERROR'   THEN 10
    WHEN level = 'WARNING' THEN  5
    ELSE 1
  END AS severity_score,
  event_ts
FROM raw_events_gen
WHERE level = 'ERROR' OR level = 'WARNING';
