Demo 01 of 04

Datagen → Filter
→ Broadcast to Two Sinks

Build your first fan-out pipeline entirely on the canvas. A synthetic data source produces order events. A filter keeps only high-value orders. Two sinks receive the same stream simultaneously — Print (Debug) shows output in the TaskManager log, and Elasticsearch indexes every record.

✓ Flink session connected ✓ No JARs for Print sink ⚠ ES JAR for Elasticsearch sink → No ES? Use Blackhole instead
This Demo What You'll Build Prerequisites Steps 1 · Open Pipeline Manager 2 · Pipeline Settings 3 · Add Datagen Source 4 · Add Filter 5 · Add Print Sink 6 · Add Elasticsearch Sink 7 · Connect the Nodes 8 · Run & Submit 9 · Verify Output Reference Generated SQL Troubleshooting

▦ What You'll Build

Three nodes, three edges. By the end of this demo you will have a running Flink job that continuously generates fake order data, filters rows where amount > 500, and fans out to two sinks at the same time using BROADCAST edges.

Concepts practised: adding operators, configuring with ✏ edit, drawing edges, changing edge types with double-click, running animation, submitting to Flink.

! Prerequisites

  • Studio connected — you have an active Flink session shown in the topbar.
  • Print sink — built into Flink. No JAR needed.
  • Elasticsearch sink — requires flink-sql-connector-elasticsearch7-x.x.x.jar in /opt/flink/lib/. Upload via ⊙ Systems → Upload JAR then restart the Gateway.
💡
No Elasticsearch? Replace the ES sink with a Blackhole sink from the Sinks palette. The fan-out structure and BROADCAST edges work identically. You'll still see data in the Print sink.
1

Open Pipeline Manager

Click ◈ Pipeline in the Studio topbar. The Pipeline Manager opens in Builder mode. Click the pipeline name Untitled Pipeline at the top and rename it:

fanout-demo

Make sure you are on the ◈ Builder tab.

2

Configure Pipeline Settings

Click ⚙ Settings at the bottom-left of the canvas. Set the following — these become SET statements in the generated SQL:

Execution Mode:          streaming
Parallelism:             1
Checkpoint Interval:     10000
Checkpoint Mode:         EXACTLY_ONCE

Close Settings.

3

Add the Datagen source

In the palette on the left, find Sources → Datagen. Drag it onto the canvas. The config modal opens automatically. Fill in every field:

Datagen Source — ✏ edit fields
Table Nameorders_gen
Rows / sec10
Schemaorder_id BIGINT customer STRING product STRING amount DOUBLE status STRING ts TIMESTAMP(3)
Watermark Colts
Watermark Delay5

Click ✓ Apply. The node badge changes from ⚠ config to ✓ ready.

4

Add the Filter transform

Drag Transformations → Filter onto the canvas, to the right of the Datagen node. Click ✏ edit on the node (or double-click it).

ℹ
Filter has one parameter only: the WHERE condition. There is no separate Table Name field. The node label is just a canvas display name — in SQL generation the filter becomes a WHERE clause appended to the INSERT INTO ... SELECT ... FROM source.
Filter — ✏ edit fields
Node Labelfilter_high_value
↑ display onlynot a SQL table name
WHERE Conditionamount > 500.0

Click ✓ Apply. The condition becomes WHERE amount > 500.0 in the generated INSERT INTO.

5

Add the Print (Debug) sink

Drag Sinks → Print (Debug) onto the canvas, below-right of the Filter node. Click ✏ edit:

Print (Debug) — ✏ edit fields
Table Nameprint_high_orders

Click ✓ Apply.

6

Add the Elasticsearch sink

Drag Sinks → Elasticsearch onto the canvas, above-right of the Filter node. Click ✏ edit:

Elasticsearch Sink — ✏ edit fields
Table Namees_high_orders
ES Hostshttp://elasticsearch:9200

Before clicking Apply, click ⊙ Test Connectivity. A green result means Elasticsearch is reachable. Then click ✓ Apply.

💡
No Elasticsearch? Skip this step and drag a Blackhole sink instead. Configure Table Name: blackhole_sink and it will silently discard records — the pipeline structure is identical.
7

Connect the nodes and set edge types

Draw three edges by dragging from the right port (●) of one node to the left port of the next:

  • orders_gen → filter_high_value
  • filter_high_value → print_high_orders
  • filter_high_value → es_high_orders

Now set edge types. Double-click the edge between filter_high_value → print_high_orders. In the edge config modal, change Type to BROADCAST and click Apply. Repeat for the edge to es_high_orders — also set to BROADCAST. The first edge can stay as FORWARD.

ℹ
Two sinks = two INSERT statements. Because there are two sink nodes, Pipeline Manager generates an EXECUTE STATEMENT SET BEGIN ... END block containing one INSERT per sink — both with the same WHERE condition applied. This runs as a single Flink job with two sink operators.

Click Layout in the toolbar to auto-arrange nodes. Click Layout then Submit.

8

Run the animation, then Submit

Click ▶ Run in the toolbar. Animated particles should flow from Datagen → Filter → both sinks. The status bar should show 3 nodes · 3 edges · ✓ Ready. Click ⏹ Stop to stop the animation.

Click Submit. Studio inserts the generated SQL into the editor and executes it. The Job Graph tab opens automatically — you should see a running job.

✓
What happens on Submit: Studio generates the full DDL + INSERT INTO SQL, opens a new editor tab, and runs it. The job name includes your session name so it's traceable in the Job Graph.
9

Open a new SQL editor tab and run these verification queries:

⟨⟩ Generated SQL

This is the full SQL that Pipeline Manager generates and submits. Because there are two sinks, it wraps both INSERT statements in an EXECUTE STATEMENT SET block — this runs as a single Flink job with two concurrent sink operators. View it at any time in the ⟨/⟩ SQL tab inside Pipeline Manager.

SET 'execution.runtime-mode'            = 'streaming';
SET 'parallelism.default'               = '1';
SET 'execution.checkpointing.interval'  = '10000';
SET 'execution.checkpointing.mode'      = 'EXACTLY_ONCE';

-- Source: orders_gen
CREATE TABLE IF NOT EXISTS orders_gen (
  order_id  BIGINT,
  customer  STRING,
  product   STRING,
  amount    DOUBLE,
  status    STRING,
  ts        TIMESTAMP(3),
  WATERMARK FOR ts AS ts - INTERVAL '5' SECOND
) WITH (
  'connector'       = 'datagen',
  'rows-per-second' = '10'
);

-- Sink: print (debug)
CREATE TABLE IF NOT EXISTS print_high_orders (
  order_id  BIGINT,
  customer  STRING,
  product   STRING,
  amount    DOUBLE,
  status    STRING,
  ts        TIMESTAMP(3)
) WITH (
  'connector' = 'print'
);

-- Sink: elasticsearch
CREATE TABLE IF NOT EXISTS es_high_orders (
  order_id  BIGINT,
  customer  STRING,
  amount    DOUBLE,
  status    STRING,
  ts        TIMESTAMP(3)
) WITH (
  'connector' = 'elasticsearch-7',
  'hosts'     = 'http://elasticsearch:9200',
  'index'     = 'high-orders'
);

-- Two sinks = EXECUTE STATEMENT SET with one INSERT per sink
-- The Filter's WHERE clause is applied to both INSERTs
EXECUTE STATEMENT SET BEGIN

INSERT INTO print_high_orders
SELECT * FROM orders_gen
WHERE amount > 500.0;

INSERT INTO es_high_orders
SELECT order_id, customer, amount, status, ts
FROM orders_gen
WHERE amount > 500.0;

END;
