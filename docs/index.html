Quickstart

FlinkSQL Studio is a single HTML file served by nginx. The fastest path is Docker Compose — it starts the Studio alongside a full Flink cluster in one command.

bash 
git clone https://github.com/coded-streams/flinksql-studio
cd flinksql-studio
docker compose up -d
open http://localhost:3030

Starts: JobManager, TaskManager(s), SQL Gateway, CORS proxy, and the Studio nginx container.

python3 -m http.server 8080
# Open studio/index.html — use Direct Gateway mode
helm repo add flinksql https://coded-streams.github.io/flinksql-kube-operator/charts
helm install flinksql-operator flinksql/flinksql-operator \
  --namespace flinksql-system --create-namespace

kubectl apply -f - <<EOF
apiVersion: codedstreams.io/v1alpha1
kind: FlinkSQLStudio
metadata:
  name: my-studio
  namespace: flink
spec:
  gateway:
    host: flink-sql-gateway
    port: 8083
  jobmanager:
    host: flink-jobmanager
    port: 8081
EOF
💡
First thing every session Open Snippets → CONFIG → ⚡ Recommended Streaming Config and click Insert. Sets runtime mode, parallelism, checkpointing, state TTL, and MiniBatch in one shot.

🏗Architecture

FlinkSQL Studio is a static HTML file served by nginx. All SQL execution happens via the Flink SQL Gateway REST API. The browser manages all session state client-side.

nginx proxies /flink-api/ to the SQL Gateway and /jobmanager-api/ to the JobManager REST API, solving browser CORS without touching Flink configuration.

⚠️
Session handles are ephemeral Container restarts invalidate session handles. Use Workspace Export to save SQL across sessions, then reconnect and re-run your CREATE TABLE statements.

✦All Features

Editor

🗂

Multi-Tab Editor

Write SQL across unlimited named tabs simultaneously. Each tab has its own editor, results, logs, and performance metrics, completely isolated.

✏️

Tab Rename

Double-click any tab to rename it (20-char limit with live counter). Name tabs by pipeline step to keep complex multi-job workflows organised.

🎨

Syntax Highlighting

Full Flink SQL highlighting — keywords, literals, comments, operators. CodeMirror-powered editor with line numbers and proper indentation.

▶️

Run Selection

Highlight any portion of SQL and click Selection to run only that statement — essential when a tab contains multiple CREATE TABLE definitions.

🔁

Format SQL

One-click auto-formatting normalises whitespace and indentation. Works on any statement type including nested subqueries and window functions.

📐

Explain Plan

Run EXPLAIN on any SELECT or INSERT to see Flink's logical and physical execution plan before submitting the job.

Results & Data

⚡

Live Streaming Results

SELECT from a Kafka source — rows appear in the Results tab every 300ms as they stream in. Tab auto-activates on first row. Badge shows live row count.

📄

Paginated Result Table

Type-aware rendering: timestamps in teal, numbers right-aligned, NULLs styled distinctly. Pagination for large result sets. Supports up to 10,000 rows in memory.

↓

Full Session Export

Downloads a combined file: query results as CSV + full session log + performance timings. Works even when results are empty.

⏹

Stop Streaming Query

Stop button appears during any running operation. Cancels the active Kafka SELECT without affecting independently running Flink jobs.

Job Graph

🔀

Real-Time DAG

Live DAG of any running Flink job pulled from the JobManager REST API. Nodes show operator type, parallelism, and record counts updating in real time.

🖱️

Zoom & Pan

Mouse wheel zooms toward cursor (20%–300%) with a % indicator. Click-drag to pan. Double-click empty canvas to reset zoom and pan to 100%.

🔍

Node Detail Panel

Double-click any node: status, parallelism, records in/out per second, backpressure, duration. Click × to close.

🚨

Fault Highlighting

FAILED/CANCELING nodes highlighted in red with animated borders. Edge colours reflect exchange type: FORWARD, HASH, REBALANCE.

✕

Cancel Job

Cancel button for RUNNING/RESTARTING jobs. Sends cancel to JobManager REST API and auto-reloads the graph to confirm state change.

🔄

Auto Job Graph on INSERT

INSERT INTO detects the Job ID response and automatically navigates to the Job Graph and loads the submitted job — no manual tab switching.

Performance Monitoring

📊

Per-Query Timing

Every statement timed from submission to first result. Performance tab shows execution duration bar charts for each query with row count and status.

📈

Live KPI Metrics

Session-wide cards: total queries, total rows returned, average duration, fastest and slowest query. Refreshes with every completed statement.

Sessions & Workspace

🛡

Multi-Session Isolation

Create multiple simultaneous SQL Gateway sessions. Each session has completely isolated tabs, results, logs, history, and performance data.

🗃

Catalog Context Tracking

Active catalog.database always shown in statusbar. USE CATALOG and USE statements update it live. Sidebar refresh no longer hijacks your context.

📋

Query History

Every executed statement logged with timestamp, status, and full SQL. Click any history item to load it back into a new tab.

💾

Workspace Export / Import

Export all tabs, SQL, and history as a named JSON file. Import on any machine or after a restart to resume exactly where you left off.

Themes

🌑

Dark Theme

Default. Deep navy with teal accents. Optimised for long sessions and low-light environments. All graph elements and highlighting adapt.

☀️

Light Theme

High-contrast clean theme for presentations and screen sharing. One-click toggle from the top navigation bar. Preference persisted in localStorage.

Session Setup — run once per session

USE CATALOG default_catalog; USE `default`;
SET 'execution.runtime-mode' = 'streaming';
SET 'parallelism.default' = '1';
SET 'pipeline.operator-chaining' = 'false';
SET 'execution.checkpointing.interval' = '10000';
1

Create Tables — DDL tab

CREATE TEMPORARY TABLE trade_gen (
  trade_id STRING, symbol STRING, side STRING,
  quantity DOUBLE, price DOUBLE, exchange STRING,
  event_time TIMESTAMP(3),
  WATERMARK FOR event_time AS event_time - INTERVAL '5' SECOND
) WITH ('connector' = 'datagen', 'rows-per-second' = '5');
2

Start Producer — own tab → Job Graph shows SOURCE → SINK

INSERT INTO trade_events_kafka
SELECT trade_id, symbol, side, quantity, price, exchange, event_time
FROM trade_gen;
3

Branching Enrichment — own tab → branching DAG in Job Graph

EXECUTE STATEMENT SET BEGIN

INSERT INTO enriched_trades_kafka
SELECT trade_id, symbol, side,
  ROUND(quantity * price, 2) AS trade_value,
  CASE WHEN quantity * price >= 100000 THEN 'CRITICAL'
       WHEN quantity * price >=  10000 THEN 'HIGH'
       ELSE 'LOW' END AS risk_level, event_time
FROM trade_events_source;

INSERT INTO branch_alerts_sink
SELECT trade_id, symbol, ROUND(quantity * price, 2) AS trade_value, event_time
FROM trade_events_source WHERE quantity * price >= 10000;

END;
4

Windowed Aggregation — own tab → 5-node Job Graph

INSERT INTO trade_summary_kafka
SELECT window_start, window_end, symbol, side,
  COUNT(*) AS trade_count,
  ROUND(SUM(quantity * price), 2) AS total_volume
FROM TABLE(TUMBLE(TABLE enriched_trades_source,
  DESCRIPTOR(event_time), INTERVAL '10' SECOND))
GROUP BY window_start, window_end, symbol, side;
5

Preview Live Data — own tab → rows stream into Results tab

CREATE TEMPORARY TABLE enriched_trades_view ( ... ) WITH (
  'connector' = 'kafka', 'topic' = 'enriched-trades',
  'properties.bootstrap.servers' = 'kafka-01:29092',
  'properties.group.id' = 'preview-consumer',
  'format' = 'json', 'scan.startup.mode' = 'earliest-offset'
);
SELECT trade_id, symbol, side, trade_value, risk_level, event_time
FROM enriched_trades_view;
💡
Always create a separate source-mode view to query live data. Tables created as sinks (no scan.startup.mode, no group.id) cannot be used in SELECT queries.

☸️Kubernetes Operator

The FlinkSQL Kube Operator manages FlinkSQL Studio deployments as first-class Kubernetes CRDs. Deploy and lifecycle-manage Studio instances with a single kubectl apply.

flowchart TD CR["FlinkSQLStudio CR\nkubectl apply"] OP["Kube Operator\nKopf controller"] DEP["Deployment\nnginx pods"] SVC["Service\nClusterIP/NodePort/LB"] POD["nginx Pod\nIDE + reverse proxy"] GW["Flink SQL Gateway"] JM["Flink JobManager"] CR-->|"reconcile"|OP OP-->|"creates / owns"|DEP OP-->|"creates / owns"|SVC DEP-->POD POD-->GW POD-->JM style CR fill:#1a2130,stroke:#00d4aa,color:#f0f6fc style OP fill:#1a2130,stroke:#9b72cf,color:#f0f6fc style DEP fill:#1a2130,stroke:#4fa3e0,color:#f0f6fc style SVC fill:#1a2130,stroke:#4fa3e0,color:#f0f6fc style POD fill:#1a2130,stroke:#3fb950,color:#f0f6fc style GW fill:#1a2130,stroke:#f0a500,color:#f0f6fc style JM fill:#1a2130,stroke:#f0a500,color:#f0f6fc

What the Operator Does

  • Watches for FlinkSQLStudio CRs (group: codedstreams.io/v1alpha1)
  • Creates and manages a Deployment and Service for each instance
  • Sets ownerReferences — deleting the CR garbage-collects everything automatically
  • Updates status.conditions with Ready: True/False for GitOps workflows
  • Reconciles on any spec change — update replicas, gateway config, or resource limits live
🐍
Built with Kopf (Kubernetes Operator Pythonic Framework) Written in Python — no Go required. The controller handles create, update, and delete events on the CRD using the Kopf framework.

🚀Deployment Options

🐳

Docker Compose

Full stack in one command — Studio, Flink cluster, SQL Gateway, Kafka. Best for local development.

docker compose up -d
📄

Standalone HTML

Serve studio/index.html from any static web server. Connect via Direct Gateway mode.

python3 -m http.server 8080
☸️

Kubernetes Operator

CRD-based deployment via Helm. GitOps-friendly status conditions. No PVC required.

helm install flinksql-operator \
  flinksql/flinksql-operator
☁️

Cloud Flink (Remote)

Run Studio anywhere. Direct Gateway mode connects to Confluent, Ververica Cloud, or AWS Managed Flink with bearer token auth.
